Platform

The Read-Only Intelligence Layer

Neraium is not a sensor platform. It is not a SCADA upgrade. It is the intelligence layer operating above existing water infrastructure — ingesting telemetry, identifying structural deviation trajectories, and delivering operational foresight to licensed personnel before failure becomes consequence.

CAP-001
Deviation Detection
Identifies sustained structural deviation patterns across multi-signal telemetry streams in real time. Distinguishes genuine drift from transient noise using trajectory analysis, not simple thresholds.
CAP-002
Operator Authority
All response decisions remain with licensed operators and building management systems. Neraium provides intelligence — never actuation. Fully compliant with utility, safety, and insurance regulations.
CAP-003
Multi-Domain Correlation
Correlates pressure, flow, thermal, and chemical signal domains simultaneously. Cross-domain patterns reveal degradation trajectories invisible to any single-sensor monitoring approach.
CAP-004
Continuous Vigilance
24/7 telemetry analysis with early warning windows opening 100+ hours before failure cascade. No manual review required for routine operations. Every anomaly is logged and traceable.
CAP-005
Sensor Agnostic
Integrates with existing BAS, SCADA, and IoT sensor infrastructure. No proprietary hardware required. Deployment does not interrupt live operations or require system modification.
CAP-006
Audit-Grade Logging
Immutable event records for regulatory compliance, insurance underwriting, and post-incident analysis. Full deviation trajectory history preserved at all detection resolutions indefinitely.

Architecture

Three-Layer Intelligence Stack

Neraium's architecture separates raw data ingestion, analytical processing, and operational alerting into discrete, independently redundant layers — ensuring clean separation between raw telemetry and actionable intelligence at every stage of the pipeline.

Layer 01
Ingest
Protocol-agnostic raw telemetry ingestion from BAS, SCADA, and IoT sources. Continuous stream normalization, timestamp reconciliation, and signal validation before analytical handoff.
→
Layer 02
Analyze
Multi-domain correlation engine processing all signal streams simultaneously. Sustained deviation trajectory modeling, baseline drift classification, and composite scoring across all monitored domains.
→
Layer 03
Alert
Prioritized intelligence delivery to operations teams and integrated CMMS platforms. Contextual deviation reports with mean 183-hour lead time — structured for immediate operator action.

Deployment

Non-Invasive Deployment Model

Neraium deploys as a read-only overlay on existing infrastructure without service interruption, proprietary hardware, or system modification. Compatible with all major building automation, SCADA, and IoT sensor platforms. Operational within days, not months.

Deployment Protocol
  • 01
    Infrastructure Survey
    Remote assessment of existing sensor architecture, data protocols, and telemetry streams. No on-site disruption required during scoping phase.
  • 02
    Integration Configuration
    Ingestion layer configured to existing BAS and SCADA outputs. Protocol adapters handle MODBUS, BACnet, MQTT, and proprietary data formats without middleware.
  • 03
    Baseline Calibration
    72-hour supervised baseline period establishes facility-specific normal operating parameters across all monitored domains. No manual configuration required.
  • 04
    Operational Handoff
    Intelligence layer goes live. Operations team briefed on alert triage protocol. Ongoing model refinement and support included in all deployment tiers.
Platform Compatibility
  • ◆
    SCADA Systems
    Full integration with Ignition, FactoryTalk, WonderWare, and open-standard SCADA platforms. Read-only data access preserves operational system integrity.
  • ◆
    Building Automation
    BACnet IP/MS-TP, LonWorks, KNX, and proprietary BAS platforms. Certified integration profiles for Johnson Controls, Siemens, and Honeywell environments.
  • ◆
    IoT Sensor Networks
    MQTT, LoRaWAN, Zigbee, and cellular-connected arrays. Edge processing available for bandwidth-constrained or air-gapped deployment environments.
  • ◆
