Technical

Read only infrastructure intelligence for deviation detection

This page describes the system boundary, analytical method, and evidence discipline. Neraium is strictly non actuating and does not modify operational systems. Outputs are designed for operator review and technical diligence.

Read only Non actuating Operator authority preserved Telemetry first
Boundary
System boundary and non actuation property
Neraium ingests telemetry and produces reviewable intelligence. It does not issue commands, trigger automated interventions, or modify system configuration. Control authority and responsibility remain with the operator.
01

Observation only

Telemetry is consumed as an input stream. Neraium does not write back into control networks.

02

No automated action

No automated maintenance actions. No setpoint changes. No valves, pumps, or actuators controlled by Neraium.

03

Operator retains authority

Operators and engineers evaluate outputs and decide how to respond within established procedures and approvals.

Method
Deviation detection on multi signal telemetry
The core method identifies sustained structural deviation in a telemetry environment. Signals are evaluated as a system rather than as isolated readings. Outputs emphasize persistence, trajectory, and operator reviewability.
Step A

Baseline window

Select a stability window to characterize normal operating behavior for the monitored system.

Step B

Distance and persistence

Compute deviation from baseline and require persistence over time to reduce noise admissions.

Step C

Admission thresholding

Admit deterioration signals only when evidence exceeds defined thresholds and stability checks.

Evidence
Evidence discipline and claim boundaries
Claims must remain bounded to evidence. Benchmarks validate analytical mechanics, not field performance. Field validation requires domain telemetry and operating constraints.
Today

Not claimed

Required

Deployment
Deployment model designed for conservative environments
Deployment prioritizes minimal disruption and clear boundaries. Neraium is designed to integrate without introducing control paths or changing operational responsibility.
01

Connect to telemetry sources

BAS, SCADA, historians, and existing sensors. Integration is read only.

02

Operate within boundaries

No write access to control networks. No automated interventions. Outputs delivered for review.

03

Validate with operators

Operators confirm whether admissions correspond to real deterioration, improving calibration and reliability.
