Technical Specification

System Boundary, Methodology, and Empirical Validation

Neraium operates as a read-only infrastructure intelligence layer. It ingests telemetry, performs multivariate drift analysis, and produces structured outputs for operator review. The system does not execute control actions or automated remediation.

01 — System Boundary
Operational Scope
The platform receives telemetry and produces analytical outputs. It does not manage equipment, alter control states, or perform automated intervention.

Read-Only Telemetry Ingestion

Receives operational signals without issuing commands downstream.

Multivariate Drift Detection

Mahalanobis-based deviation modeling across correlated parameters.

Structured Evidence Output

Preserves analytical state for review and governance documentation.

02 — Methodology
Benchmark Analytical Framework
Validation performed using 30-cycle baseline modeling, Mahalanobis multivariate drift computation, 7-cycle smoothing, 2-hit admission logic, and a 4σ threshold.
